## 执行流程

### 数据兜底策略（优先级顺序）

若某数据源失败，自动降级至下一级，层层兜底。

| 优先级 | 数据源 | 说明 |
|--------|--------|------|
| P1 | 天天基金官方GSZ API | 批量获取盘中估算净值（最快、最准） |
| P2 | 新浪基金实时接口 | 单只查询，格式 `f_{code}`，返回实时涨跌幅 |
| P3 | AkShare 基金实时 | `ak.fund_industry_hist_open()`, `ak.fund_open_fund_info()` |
| P4 | Tushare 基金估值 | `ts.fund_nav()` 获取近期单位净值走势 |
| P5 | 重仓股穿透 | 基于季报持仓 + 实时行情计算 |
| P6 | 昨日NAV兜底 | 用最近确认净值 + 0 涨跌幅，标注「数据暂不可用」 |

**兜底触发条件**：某基金本轮任一接口返回空/HTTP非200/解析失败，即触发降级。

---

### P1：获取官方估值

调用天天基金 API 批量获取全量持仓的盘中实时估算净值（GSZ）：

```
GET https://fundmobapi.eastmoney.com/FundMNewApi/FundMNFInfo?pageIndex=1&pageSize=50&plat=Android&appType=ttjj&product=EFund&Version=1&deviceid=openclaw-agent-engine&Fcodes={基金代码列表,逗号分隔}
```

解析字段：NAV（昨日净值）、GSZ（盘中估算）、GSZZL（估算涨跌幅）、GZTIME（更新时间）。

**若 GSZ 有效**：
- 单只预估收益 = 份额 × (GSZ - NAV)
- 单只预估涨跌幅 = GSZZL%

**若 GSZ 无效或缺失**：降级至 P2。

### P2：新浪基金实时接口

单只基金查询（支持批量逗号分隔）：
```
GET https://hq.sinajs.cn/list=f{基金代码}
```

解析：返回内容含 `fdatas` 字段，取最新估算涨跌幅。若解析失败或数据为空，降级至 P3。

### P3：AkShare 基金接口

```python
import akshare as ak

# 盘中实时估值（部分基金支持）
ak.fund_industry_hist_open(date=today)  # 行业估算数据
ak.fund_open_fund_info(fund_code, indicator="近1月")  # 净值走势
```

若返回有效数据，取最近估算涨跌幅；若失败，降级至 P4。

### 重仓股穿透（官方GSZ不可用时）

触发条件：P1-P4 均无法获取有效数据。

1. **获取重仓股**：
   ```
   GET http://fundf10.eastmoney.com/FundArchivesDatas.aspx?type=jjcc&code={基金代码}&topline=10&rt=0.618
   ```
   解析：股票代码 + 占净值比（已是股票总仓位 × 该股权重，无需再乘仓位）

2. **批量获取实时行情**：
   将股票代码转为东方财富 secid 格式（沪市 1.代码，深市 0.代码，港股 116.代码）：
   ```
   GET https://push2.eastmoney.com/api/qt/ulist.np/get?fltt=2&invt=2&fields=f2,f3,f12,f14&secids={逗号分隔}
   ```
   备用（限流时切）：`https://hq.sinajs.cn/list={sh/sz+代码}`

3. **穿透公式**：
   ```
   预估收益 = 份额 × NAV × Σ(重仓占净值比% × 重仓实时涨跌幅%)
   覆盖率 = Σ(前十大占净值比%) ÷ 基金股票仓位%
   ```

### 第三步：汇总输出

计算完成后，排序输出飞书卡片。

## 注意事项

- **覆盖率门槛**：穿透计算时，若前十大覆盖率 < 50%，在卡片中提示"持仓分散，估算仅供参考"
- **QDII处理**：001668（汇添富全球移动互联）和 118001（易方达亚洲精选）为海外基金，无盘中数据时标注"暂无盘中估值，参考基金净值趋势"
- **去重逻辑**：汇总重仓股TOP3时，同一股票被多只基金持有需合并计算影响金额
- **时间判断**：仅在交易日盘中（9:30-15:00）执行；午间休市（11:30-13:00）使用最近午盘数据；非交易日提示用户
