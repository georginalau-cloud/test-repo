# VAU Writer Skill

帮人写VAU（Vision-Action-Update）的技能。

## 预设角色

当用户说"帮我写VAU"且未提供角色说明时，默认使用以下角色：

**沛柔**
- 岗位：采购BP（Business Partner）
- 直接上级：周宇（BP团队head，Line Manager）
- 跨级领导：Peter Hu（VP，商业资源管理部门head）
- 服务部门：FY品牌全部门、数字化发展、用户数字化产品、自动驾驶、智能硬件、海外（均为平行一级部门）
- 角色性质：个人贡献者，无团队
- 特殊边界：
  - Vision直接照搬直接上级
  - Action不越权：团队建设、HR工作、要求别人做到的不承接
  - AI4P核心成员，任务轻量化（参与而非主导）
  - 跨境采购表述不用"合规"等词，用"协同""流程拉通"

> 如用户说"帮别人写"，则临时询问该人的岗位、直接上级、服务部门、角色性质。

---

## 核心流程

### Step 1：解析文档
识别所有人物的Vision和Action，区分层级：
- 直接上级（Vision可以直接照搬）
- 跨级领导（Vision可以照搬，Action选相关）
- 平级/其他部门（不照搬Vision，相关内容融入自己的Action）

### Step 1.5：追问季度重点工作
在推导Action之前，主动问用户：
> "这个季度你有没有什么重点工作想放进VAU？比如想主攻的方向、想突破的领域、或者上级私下交代的任务。"

原因：VAU不只是分解上级的意图，也需要体现自己的节奏。如果用户有明确想法，在Vision 1的Action里优先体现；如果没有，继续走文档推导流程。

### Step 2：逐条推导Action归属

对每个Action问三个问题：

**Q1：这个Action的职能是不是我的？**
不是 → 过滤掉

**Q2：这个Action的描述有没有越权？**
越权特征 → 改写为"参与""配合"或"自己做到"：
- "打造/建立团队XXX" → "参与/自己做到"
- "完善能力模型/能力画像" → "对照提升自己"
- "要求别人做到" → "自己做到"
- "统筹/主导XXX"（不该你主导的）→ "协同/支持"

**Q3：这个Action的表述有没有风险？**
- "确保XXX落地" → "协同保障"
- "不合规/推进合规" → "流程拉通/配合流程"
- "消除风险" → "配合识别"

### Step 3：融入跨部门提及

其他人在文档里提到你的内容（如@你的话）：
- 改写成符合自己角色的版本
- 落脚点是"自己能做到什么"
- 不要原文照搬（比如别人是要求全团队，你只承接"自己做到"）

### Step 4：检查任务量和时间节奏

- 如果Q2所有Action都能做完，Q3没得写 → 加时间节奏："Q2完成框架，Q3深化"
- 每个Vision配2-4个Action（常见3个）
- 整体任务量评估：Q2做完的基础上，Q3是否有递进空间

### Step 5：中英双语输出

## 越权Action速查

| ❌ 越权说法 | ✅ 安全改写 |
|---|---|
| 建立/打造团队XXX | 参与团队XXX |
| 完善能力模型/能力画像 | 对照模型识别自身短板 |
| 要求别人AI普及 | 自己AI使用达100% |
| 确保海外合规落地 | 与海外团队协同保障流程 |
| 统筹/主导（不该你主导的） | 协同/支持 |
| 消除风险 | 配合识别风险点 |

## 风险表述速查

| ❌ 踩雷说法 | ✅ 安全说法 |
|---|---|
| 确保落地 | 协同保障 |
| 不合规 | [直接不用这个词] |
| 合规推进 | 流程拉通/配合 |
| 风险消除 | 风险识别/配合管控 |
| 负责XXX | 承接/负责跟进XXX |
