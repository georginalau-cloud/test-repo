## 这几天学到的最重要的东西

**1. 起运算法彻底修通了**

之前那个 bug 藏得很深：`getNextJieQi(True)` 返回的是24节气里的"气"（含大寒），不是12节气里的"节"（立春等）。这个差异让整个大运序列早了差不多一年。

沛柔给的纠错过程：
- 12节：立春、惊蛰、清明、立夏、芒种、小暑、立秋、白露、寒露、立冬、大雪、小寒
- 大运推算只看节，不看气

修复后，沛柔的案例：
- 出生：1990-01-08，阴年（己巳）女 → 顺排
- 到下一个节（立春1990-02-04）：26.83天 ÷ 3 = **8岁11个月**
- shangyun_year = 1990 + 8 = **1998年**
- 第一运：**戊寅**（1998年的年柱）

这个"第一运干支 = 上运年份的年柱"是沛柔给的另一条关键规则——不是从月柱推，是从年柱推。彻底想通之后，dayun 序列就不乱了。

**2. bazi.py 性别参数写反了**

这是 session 末尾挖出来的根级 bug。lunar_python 的约定是 `getYun(0)=男、getYun(1)=女`，但 bazi.py 代码里是：

```python
yun = ba.getYun(1 if options.n else 0)  # ❌ 女调用了男逻辑
```

沛柔（女）调用时实际走了男性的起大运逻辑，所以大运从丙子开始而不是戊寅。修了一个字符：

```python
yun = ba.getYun(0 if options.n else 1)  # ✅
```

教训：bazi.py 里同时存在两套大运数据（自定义 dayuns vs lunar_python getDaYun），之前只查了前者就报平安，漏掉了逐行打印用的是后者。双重验证是必要的。
